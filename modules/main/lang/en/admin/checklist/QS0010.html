The factory settings of the Bitrix Framework standard modules may not always be in accordance with the website specifications, which is why these settings are a subject to thorough examination and possible alteration.

Assume you need to configure the Kernel module (Settings > System Settings > Module Settings > Kernel). The most commonly changed parameters are as follows.

  1. Shared authorization. Do you need to spread authorization and authorization cookies across all websites running on this instance of Bitrix Framework? Check it here: Settings > System Settings > Websites > Websites. You didn't forget to delete your test domains, do you?
  2. Is there a correct default e-mail address (a.k.a. default sender address) specified?
  3. Can the website templates be edited in the visual editor?
  4. Do the website specifications allow visitors to self-register? Do we need to use CAPTCHA? Does an e-mail address need to be unique?
  5. Check the the storage times of system objects match the web project and hardware capacity (the "Days to keep ...." parameters).
  6. Ensure the Bitrix Framework updates are checked daily.

Inspect the settings of each of the other system modules.

  • Make sure the module settings in Settings > System Settings > Module Settings are checked and match the website specifications.
